## 🤖 AI Agent

• 🔥13 **LangChain Anthropic 1.4.4 修正跨 provider tool-call ID 正規化** — `langchain-anthropic==1.4.4` 修掉跨供應商 tool-call ID 正規化問題，並讓 integration tests 在 transient failures 下 retry；這種修補對 agent workflow 很關鍵，因為 tool call ID 一旦不一致，串流、重試與追蹤就可能出現難查的狀態錯亂。風險面是 adapter 層越厚，使用者越需要針對自家 provider matrix 做回歸測試。
  📎 來源：[GitHub](https://github.com/langchain-ai/langchain/releases/tag/langchain-anthropic%3D%3D1.4.4)

• 🔥13 **OpenClaw 連續釋出 alpha / beta，聚焦 agent runtime 復原能力** — OpenClaw 5月29日發布 `v2026.5.29-alpha.1`，前一個 beta 說明提到 subagent cwd/workspace 分離、hook context prompt-local、session lock timeout abort 後釋放，以及避免 stale restart continuation。這些都是 agent 系統從 demo 走向長任務時會遇到的工程問題；批評視角是 alpha/beta 節奏快，部署者要更保守地釘版本。
  📎 來源：[v2026.5.29-alpha.1](https://github.com/openclaw/openclaw/releases/tag/v2026.5.29-alpha.1) | [v2026.5.28-beta.3](https://github.com/openclaw/openclaw/releases/tag/v2026.5.28-beta.3)

• 🔥10 **Datasette 1.0a31 把資料庫寫入與 stored queries 納入權限模型** — Datasette alpha 版新增具備權限者可執行 SQL write queries，並把 canned queries 改名為 stored queries，支援私有或團隊共享。它不是典型 AI agent 新聞，但對 agentic data tools 很重要：如果自然語言資料代理要進入真實資料庫，權限、審計與可保存查詢會比生成 SQL 本身更關鍵。
