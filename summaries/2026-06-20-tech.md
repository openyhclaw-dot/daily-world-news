## 🧠 LLM

• 🔥13 **PyTorch Dynamo RangeVariable 專門化 symbolic range bounds** — PyTorch viable/strict release 納入 Dynamo RangeVariable 的 symbolic range bound specialization，目標是讓編譯器在動態 shape 與範圍推論上更精準。這類底層編譯修補會影響 LLM training/inference framework 的穩定性，但仍屬 strict pipeline，生產採用要等穩定版與下游框架驗證。
  📎 來源：[pytorch/pytorch](https://github.com/pytorch/pytorch/releases/tag/viable%2Fstrict%2F1781898941)

• 🔥13 **PyTorch 新增 distributed backend implementation accessor** — PyTorch release 加入 `torch.distributed.get_backend_impl`，讓研究者或 framework author 可取回 underlying process group backend implementation，方便自訂 backend 實驗。批評視角是，文件也明示這些 backend method 會繞過 `torch.compile` tracing 與其他 hooks，且 API 實驗性高，企業使用要小心相容性。
  📎 來源：[pytorch/pytorch](https://github.com/pytorch/pytorch/releases/tag/viable%2Fstrict%2F1781856928)

• 🔥10 **Datasette Apps 讓自訂 HTML 應用跑在 Datasette sandbox iframe 內** — Simon Willison 推出 `datasette-apps` plugin，讓 self-contained HTML/JavaScript app 能嵌在 Datasette 內，並透過 JavaScript 對資料庫執行查詢。這對資料工具與 LLM-assisted data app 很實用；風險是 sandbox 權限、write query 控制與資料外洩邊界必須被清楚設計。
  📎 來源：[Simon Willison](https://simonwillison.net/2026/Jun/18/datasette-apps/#atom-everything)
