先進新事區。今天最值得花時間的，是 Trump-Xi 這場北京峰會的議程大翻盤。

5月14到15日這場高峰會，原本中美兩邊預設的主菜是貿易再平衡跟稀土供應鏈，所有人都以為這是一場貿易談判，順便聊聊台灣。結果伊朗戰爭打到現在，把這個盤子整個掀翻。Bloomberg 昨晚引述中方消息確認，會議議程已經正式調整，伊朗議題被推到第一位。

美方的算盤其實很單純，白宮顧問 Stephen Miller 在 CNBC 講得很白：「中國採購伊朗石油，等於替德黑蘭的預算續命」。Trump 政府希望北京切斷對伊朗的石油採購，再加上美方擔心中國對伊朗的潛在軍備援助。換句話說，美方要把伊朗當成槓桿，逼中國表態。

但中方根本不接這個球。中國外交部副部長馬朝旭把話講死，他強調「台灣問題是核心利益，必須是會談中心」，同時對美方「干涉中俄、中伊正常往來」表達嚴正立場。北京的策略很清楚，把伊朗、台灣、貿易三件事綁在一起，逼 Trump 一次性切蛋糕。如果你要我停止採購伊朗石油，那台灣議題你必須讓步；如果你要稀土，那關稅你得退一步。這是把單一議題談判，變成全局籌碼戰。

而伊朗方面也不甘寂寞。伊朗外交部 Esmaeil Baghaei 昨天放話，「任何旨在切斷我們石油出口的密謀都將反噬其發起者」。這句話的潛台詞，是 IRGC 司令 Salami 同期對紅海航運的威脅——伊朗在告訴中國，你如果鬆手，我就動荷莫茲海峽。所以中國其實也被綁住，因為地緣現實不允許它輕易翻臉。

這場峰會還有一個很多人沒注意到的細節，這是 Trump 第二任後首度國事訪華，也是2017年以來首次。象徵意義跟議程張力，都被中東打到變了味道。CFR 跟 CSIS 兩家智庫的最新分析都認為，中國在這場峰會擁有主場優勢，因為美方有太多事情需要中方配合，從伊朗石油、北韓制裁，到 AI 出口管制，每一張牌都需要北京點頭。

我的看法是，這場峰會不會有實質突破，但它會釐清一個重要訊號：Trump 政府願意為了結束伊朗戰爭，付出多少對中讓步。這個答案會決定接下來六個月，從台灣到南海到稀土，整個印太局勢的張力底色。

接下來第二個新事區的故事，是停火失敗的烏克蘭。

Trump 5月8日宣布的72小時停火，從5月9日打到5月11日，今天清晨正式屆滿。然後雙方互相把對方告上 OSCE，烏方國防部統計烏軍違規 16,071 次、俄軍對等回應，這個數字本身就證明這個停火從第一天起就是一場戲。烏外長 Sybiha 拿出俄羅斯在哈爾科夫跟赫爾松地區的無人機跟飛彈攻擊紀錄，攤在桌上要求問責。

Washington Post 昨天有一句評論點得很到位，「沒有監督機制、沒有爭端解決流程的停火，註定走向今天這個結局」。這句話幾乎可以套到 Trump 上任以來嘗試過的所有外交速食方案。

白宮發言人 Karoline Leavitt 還在嘴硬，說停火「為下一階段談判奠定基礎」，但這個論述連美國主流媒體都不買單。基輔這邊 Zelensky 在 Telegram 發文，「俄羅斯選擇了戰爭，而不是和平」，再次喊話北約要加碼武器援助。莫斯科 Peskov 則把鍋丟回基輔，暗示「特別軍事行動」進入新階段——翻譯過來就是，我們要繼續打。

值得追蹤的是接下來四十八小時，因為 EU 對俄「第二十號制裁包」連同九百億歐元烏克蘭貸款，本週四，也就是5月14日，會在布魯塞爾正式簽署。所以烏克蘭其實是在停火失敗的同時，等到了歐洲的硬通貨支援。九百億歐元分四年撥付，初步用在財政缺口跟軍工採購。這筆錢落地，等於告訴克里姆林宮：你的消耗戰邏輯，歐洲打算硬撐到底。

第三段新事，回到中東。以黎戰事打到第七十一天了，這個數字本身就讓人疲倦。黎巴嫩公衛部最新公布的累計傷亡是 2,846 人死亡、8,693 人受傷，自從 4月16日「停火」生效以來，還新增了 552 人死、1,149 人傷。這個「停火」兩個字現在只能加引號使用。

週末以軍轟炸 Nabatieh 省的發電廠，讓整個黎南陷入大規模停電。Hezbollah 同日對 Khiam 跟 Deir Siryan 的以軍陣地發射攻擊無人機，IDF 隨後擊斃一個火箭發射小組。雙方都在升級。

最值得記下來的一句話，是 IDF 參謀總長 Eyal Zamir 公開下令，「不計代價對抗 Hezbollah 無人機威脅」，並授權空軍動用儲備彈藥。這是2026年以來 IDF 第一次用這麼直白的措辭升級空中作戰授權。

貝魯特方面，Mikati 總理把以方的電廠攻擊定性為「違反日內瓦第四公約」，這個說法不是隨口講的，因為民用基礎設施在國際法上有明確的保護地位，他這是在替未來的國際法訴訟鋪路。

100 萬以上的居民——佔黎巴嫩人口五分之一——已經流離失所。重點是現在進入第七十一天，國際社會的注意力已經被烏克蘭跟伊朗瓜分，黎巴嫩這條線在媒體版面上正在被擠壓。這是一場「被遺忘的戰爭」進行式。

科技新事區，今天最值得花時間的是 SpaceX Starship Flight 12。

今晚美東時間下午五點半，UTC 二十二點三十分，Starship 第十二次飛行測試會從 Starbase 第二發射台升空。這次有三個「第一次」：第一次飛 V3 上級、第一次用全套 Raptor 3 引擎、第一次從 Pad 2 發射。Booster 19 跟 Ship 39 堆疊起來高四百零八呎，比 V2 高大約四呎，但近地軌道酬載突破百噸，幾乎是 V2 那三十五噸的三倍。

但這次任務刻意不去挑戰 Flight 11 達成過的塔架抓回。booster 跟 ship 都會選擇濺落，這是 SpaceX 罕見的「策略性退一步」——他們現在最在意的是把新架構驗證起來，不是炫技。

技術層面 Raptor 3 的關鍵突破是減少零件數、簡化管路、把推力提升到一定區間，但代價是每一台引擎的可靠性都還沒在真實飛行中得到驗證。今晚就是 33+6 共 39 台 Raptor 3 一次性的大考。

這場發射的意義超越單一任務。Anthropic 跟 SpaceX 上週才簽下三百兆瓦算力協議，現在 V3 又要把單次發射酬載翻三倍——SpaceX 正在從「商業火箭公司」變成「AI 算力的硬體基礎建設供應商」之一。

科技新事第二則，LangChain 正式進入 1.x 時代。

`langchain-core` 1.4.0 昨天釋出，這不是版號小躍進，是整個套件家族從 0.3.x 跳到 1.x 主線的分水嶺。同日 LangChain Classic 1.2.18 撤回上一版的 `ls_agent_type` tag，並開始 deprecate `hub` 模組。整個 LangChain 套件家族從本週起進入「1.x 主線、0.3.x 維護」二元化階段。

對 agent 開發者來說，這代表三件事：第一，ABI 開始穩定，可以放心把 1.x 寫進生產；第二，舊的 0.3.x 程式碼需要逐步遷移，import 路徑會變；第三，LangGraph 跟 LangChain Classic 的責任邊界仍然沒理清，社群已經連續三版抱怨遷移成本。所以「穩定」是相對的，遷移路徑還很模糊。

另外有一個資安點要注意，langchain-core 之前修過的 CVE-2026-34070 路徑穿越漏洞，正式合併進 1.4.0 主線。如果你還在 0.3.86 之前的版本，這週就升級。

科技新事第三則，OpenAI 今年第七起併購。

OpenAI 收了 agentic 軟體工程平台 Hiro，這是2026年以來第七起併購；同一個月內，OpenAI 還吞下了 TBPN——Technology Business Programming Network，這是公司有史以來第一次併購媒體公司。

一邊買工具，一邊買聲量。跟 Anthropic 上週公布的 SpaceX 三百兆瓦、二十二萬張 GPU 算力協議形成兩種策略對照：Anthropic 走「算力霸主」路線，OpenAI 走「生態圈+輿論場」路線。

媒體併購這部分很敏感。當一家擁有最大語言模型市佔的公司，同時擁有產品、運算跟評論平台，獨立的科技新聞報導空間會被壓縮。今年下半年如果美國 FTC 開始調查，我不會意外。

對照組是 Cohere 跟德國 Aleph Alpha 宣布合併。這是歐洲跟北美兩家「非 hyperscaler 派」LLM 公司的合體，繼 Mistral-DeepMind 算力協議之後，另一個對抗 OpenAI 跟 Anthropic 雙寡頭的歐洲嘗試。

科技深度區，今天我給 Simon Willison 引述 James Shore 的那段討論。

這段為什麼進深度區？因為它符合「產業典範變化」——AI agent 訂閱定價模型，今年起從「按 token 計費」轉向「按節省維護工時計費」。Anthropic 跟 OpenAI 都開始在企業合約裡綁定這個邏輯。

Shore 的原文這樣寫：「你的 AI coding agent 必須降低你的維護成本。不是降一點點。你寫程式快了兩倍？那你最好把維護成本砍一半。三倍生產力？那維護成本要降到三分之一。否則你只是用永久的維護債務，換暫時的速度加成」。

這個數學論點，正在被矽谷兩大實驗室反向採用。今年起 Anthropic 跟 OpenAI 的企業合約都不再單純按 token 收費，而是綁定「節省的維護工時」作為計價單位。這個轉變比表面看起來大很多——它把 LLM 從「按產出計價的工具」，變成「按結果計價的代理人」。

但反方有人，Andrew Quinn 反駁這個論點過於工業時代，他主張 LLM 真正帶來的是「throwaway code」文化，maintenance cost 概念本身需要重寫。如果今天九成的程式碼是一次性的、用完即丟的、為了實驗某個假設而存在的，那麼維護成本根本不應該存在於這個對話裡。

兩邊都有道理。Shore 的觀點適用於 enterprise SaaS、銀行系統、政府專案這些「程式碼活十年」的場景；Quinn 的觀點適用於 prototype、PoC、個人專案這些「程式碼活十天」的場景。LLM 的真正威力，在於它讓「程式碼可棄性」變成一個可選參數。

我的看法是，這場辯論決定的不只是怎麼收費，而是接下來五年「軟體」這個概念本身的定義。如果軟體變得可拋棄，那麼版本控制、依賴管理、技術債這整套職業，要重新發明。
