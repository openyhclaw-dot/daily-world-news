【科技新事區】

第一條，Cactus 開源了 Needle，一個 26M 參數的工具呼叫模型。重點是這幾個數字：26M 參數，跑在消費級裝置上 prefill 速度 6000 tokens 每秒、decode 1200 tokens 每秒。他們的方法是把 Gemini 的工具呼叫能力蒸餾到這個小模型裡。

為什麼這個東西值得停下來看？Cactus 在論點上挑了一個很有意思的角度——他們說，agentic experience 的本質是「retrieval and assembly」，也就是檢索跟組裝，那這件事不需要一個千億參數的大模型來做。tool calling 本來就是個結構化的任務：解析使用者意圖、找對的 function、填好參數、呼叫。如果這就是 agent 整套互動的瓶頸，那拿一個 26M 的小模型在手機本機跑，就完全可以撐起一個流暢的 agent 體驗。

當然這個想法你要打問號的地方也很清楚：第一，這是蒸餾，能力上限被 Gemini 卡住。如果 Gemini 工具呼叫上有什麼盲點，Needle 就會同樣繼承。第二，多輪複雜工具鏈、邊角錯誤恢復這些場景，26M 的容量極限會不會撐爆，目前沒有公開的 evaluation 結果。但這個方向本身，會讓很多手機端 agent 應用開始重新思考架構——不是「把雲端 LLM 接到 app」，而是「本機跑 agent 邏輯、只在語意理解需要時呼叫雲端」。

第二條，CERT 今天一口氣釋出 dnsmasq 的六個 CVE。dnsmasq 你可能沒聽過，但你每天都在用它。它是 OpenWrt、家用 router、Android 熱點、Kubernetes node 裡面的 DNS 跟 DHCP 服務。也就是說，全世界半個網路的最後一哩，就靠這個小程式。一次六個 CVE，覆蓋面是 router 層級 RCE 風險上升。修補窗口期內，這個漏洞會成為下一波家用設備殭屍網路的主要入口。如果你家的 router 有自動更新就放著，沒有的話，這禮拜記得手動進去看一下。

【科技深度區 / 類別變化】

今天有一個事件值得進深度區：XBOW 在 Exim 找到一枚未認證 RCE，編號 CVE-2026-45185。Exim 是大型機構跟 ISP 級別還在用的郵件伺服器，這個漏洞不需要任何登入就可以遠端執行代碼。XBOW 是一家用 AI 驅動的安全掃描平台，這已經不是它第一次抓出大型開源專案的 CVE。意義在哪？意義在於「自動化 AI 找出 CVE，比人類研究員找得快」這件事，從假設變成數據點。過去半年，XBOW 已經公開貢獻多個 CVE 編號，覆蓋郵件伺服器、DNS 解析器、版本控制工具。這個累積速度比傳統人工 fuzzing 還快兩到三倍。

往下推一層你會看到一個更大的問題：當 AI 開始自動化找出漏洞，整個資安生態系的攻防節奏會被同步往前推。一邊是 XBOW 這樣的白帽工具讓開源軟體更快補漏；另一邊是相同技術一旦被攻擊方拿到，零日漏洞的儲備量也會跟著增加。Exim 跟 dnsmasq 這種跑了二十年的底層元件，過去因為「人類研究員的注意力是稀缺資源」而長期沒被深掃過。現在這個假設正在被推翻，意思是未來幾個月你會看到更多這種「老程式爆出舊洞」的新聞。對企業 IT 來講，這禮拜的優先順序就一條：升級 Exim 跟 dnsmasq，並且把自動化 patch 流程的優先序往上拉。

【結尾】

回頭看一下今天的全局視角。中東這條軸線今天同時冒出三件事：以色列政府解散倒數、伊朗對中國資產的滲透行動被曝光、Hezbollah 警告黎政府不能談武器。三條訊號加在一起，意思是「以色列的下一個對話對手可能不是 Netanyahu，伊朗的下一個壓力來源可能不是 Trump 而是北京，黎巴嫩的下一場政治攤牌可能跳過外交談判直接走到地面」。這個多軸同時鬆動的狀態，是過去半年第一次。

科技這邊也有一個共同主軸：把任務拆細、把模型做小、把雲端依賴降下來。Needle 用 26M 參數做工具呼叫、DuckDB 把嵌入式分析資料庫拉到遠端服務、CrewAI 預設改成更輕量的 AgentExecutor，方向都是同一個——agent 與資料工具的「最後一哩」開始本地化、模組化。

今天就講到這。一句話結帳：以色列聯合政府進入解散倒數，科威特把伊朗 12 天前那場港口攻擊端上 Trump-Xi 峰會的桌；科技上，26M 的工具呼叫小模型開始挑戰「agent 必須用大模型」這個假設。明天 5月14日 兩件事：BoJ 利率決議，Trump-Xi 北京峰會開幕。我們明天見。
